Blasters Documentation

Introduction

Welcome to the documentation for the BlasterModule used in the game FORCE on Roblox. This system allows players to equip, use, and manage blasters within the game, enabling complex interactions like shooting, reloading, and more.

To use this module, you must first require it in your code:

local BlasterModule = require("BlasterModule")

The module provides functionality to integrate blaster tools with player characters, and allows fine control over the behavior of each blaster.

Class BlasterClient

Properties

The BlasterClient class contains various properties that define the blaster's associations and behavior for the player. Below are the key properties:



Player: Player

The player who owns the blaster. This links the tool to the player's game instance, allowing the blaster to react to player actions such as equipping or firing.



Character: Model

Represents the player's character model. The blaster is linked to this character, allowing the tool to interact with and affect the player directly.



Tool: Tool

The specific tool associated with the blaster. This property is essential for managing the functionality of the blaster in the game, including firing, reloading, and equipping the tool.



BlasterRemote: RemoteFunction

Handles communication between the server and client. This allows remote function calls to trigger actions related to the blaster.



Recoil: RecoilConstructor

Handles the recoil effects when the blaster is fired, adjusting the player's camera or movement accordingly.



BlasterStats: BlasterStats

Represents the stats associated with the blaster, such as maximum ammo, exhaust, and cooldown behavior.



BlasterIndex: Number

A unique index for the blaster instance. This is used to identify the blaster in the game.



Signals:

Signals are event-driven responses used to trigger actions based on specific events related to the blaster's state. Below are the key signals associated with the blaster:

OnEquipped Triggered when the blaster tool is equipped to the player's character.
OnUnequipped Triggered when the blaster tool is unequipped from the player's character.
OnExhausting Triggered at the start of the blaster's depletion, allowing for animations or cooldown effects to occur.
OnExhausted Triggered when the blaster's ammunition or power is depleted.
OnReconcile Triggered after a server reconciliation, ensuring data consistency across the game.
OnBulletReflected Triggered when a bullet fired from the blaster is reflected, offering a chance to handle specific game interactions.
OnBulletFired Triggered after a bullet is fired from the blaster, enabling game logic related to bullet behavior.
OnShot Triggered when a shot is fired from the blaster, allowing other game systems to react to the event.
FirerateEnded Triggered when the cooldown period for the blaster's fire rate ends, allowing it to shoot again.


Methods



BlasterClient.new

Instantiates a new BlasterClient object, which is used to manage a blaster for a specific player and their character.

Parameters

character: Model The character model to associate with the blaster.
tool: Tool The specific tool object representing the blaster.
blasterStats: BlasterStats? (Optional) Custom statistics to define specific attributes or behaviors for the blaster.

Returns

blasterClient: BlasterClient Returns the new BlasterClient object that is now managing the blaster.


BlasterClient:Shoot(position)

Handles the logic for shooting the blaster. It checks conditions like whether the blaster is equipped, whether it's exhausting, and manages burst firing.

Parameters

position: Vector3 | function The position to shoot at, or a function that calculates the position dynamically based on the current player state.

Returns

success: boolean Returns a boolean 
-- Trigger a shot at the given position or calculate a position
-- if position is a function.
local success = BlasterClient:Shoot(position)


BlasterClient:Exhaust(ping)

Handles the blaster's exhaust state, which occurs after shooting. It starts an exhaust process and sends server requests if needed to synchronize the state.

Parameters

ping: Number The ping time for the server communication, affecting how quickly the exhaust process is synchronized.

Returns

success: boolean Returns an boolean 
-- Activate exhaust effect and sound, reducing the exhaust level.
local success = BlasterClient:Exhaust(ping)


BlasterClient:Destroy()

Cleans up and removes all references to the blaster, disconnecting signals, and resetting the state of the object.

Parameters

None This method does not take any parameters.

Returns

None This method does not return any value. 
-- Destroy the current blaster instance and clear all connections.
BlasterClient:Destroy()


Private Functions



BlasterClient:_createBlasterConnections()

Sets up connections for the blaster's events, including equip/unequip, shot firing, exhaust, and bullet-related events.

Parameters

None This method does not take any parameters.

Returns

None This method does not return any value. 
-- Initialize blaster connections such as equip, shoot, and exhaust events.
BlasterClient:_createBlasterConnections()


BlasterClient:_fireBullet(startPosition, direction, startTime, isReflection)

Fires a bullet in a given direction from a start position. It handles the bullet's flight path, collision detection, and reflection logic if isReflection is true.

Parameters

startPosition: Vector3 The starting position of the bullet in the game world.
direction: Vector3 The direction the bullet should travel after being fired.
startTime: Number The time the bullet was fired, used for time-based calculations or effects.
isReflection: Boolean A flag indicating whether the bullet should be reflected (true) or not (false).

Returns

bullet: Bullet Returns a Bullet object representing the fired bullet, which can be further manipulated or tracked. 
-- This method sends bullet fire info and handles the bullet's
-- collision detection, including reflections and hit effects.
local bullet = BlasterClient:_fireBullet(startPosition, direction, startTime, isReflection)


BlasterClient:_shotEnded()

Ends the shot, clearing any ongoing shot information and firing the FirerateEnded signal.

Parameters

None This method does not take any parameters.

Returns

None This method does not return any value.


Code Examples

local BlasterClientConstructor = require("BlasterClient") 
local BlasterClient = BlasterClientConstructor.new(character, tool, blasterStats)
